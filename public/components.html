Example

<script type="module">
  import define from 'https://unpkg.com/synergy-web-components@0.4.1';

  let count = 0;

  const factory = ({
    expanded = false,
    title,
    disabled = false,
  }) => {
    return {
      id: `drawer-${count++}`,
      title,
      expanded,
      disabled,
      toggle() {
        this.expanded = !this.expanded;
      },
    };
  };

  factory.observedAttributes = ['expanded'];

  define('x-drawer', factory);
</script>
<template id="x-drawer">
  <style scoped>
    button {
      all: inherit;
    }
  </style>
  <h3>
    <button
      id="{{ id }}"
      disabled="{{ disabled }}"
      aria-expanded="{{ expanded }}"
      onclick="toggle"
    >
      {{ title }}
    </button>
  </h3>
  <div
    hidden="{{ !expanded }}"
    aria-labelledby="{{ id }}"
  >
    <slot></slot>
  </div>
</template>

In the example above, we create a new x-drawer Custom Element that can be used anywhere on the page, like so... 

<x-drawer title="foo"></x-drawer>

Initial Attributes

An elements initial attribute keys and values will be passed to your factory function during initialisation.

Observed Attributes

Observed attributes are special attributes on your component that you expect to change after initialisation. Your components view will update automatically whenever any of its observed attributes change. 

const factory = (props) => props;

factory.observedAttributes = ['name'];

Shadow Dom

Synergy doesn't support Shadow DOM at the moment. This is to preserve the pre-rendering / hydration capabilities of Synergy. There are open proposals for the Web Platform that intend to make Shadow DOM declarative and serialisable. Until then, SWC provides a polyfill for Slots, as well as a limited CSS scoping mechanism.

Slots

Even though Synergy doesn't support Shadow DOM, it does polyfill (slots)[https://developer.mozilla.org/en-US/docs/Web/HTML/Element/slot], so you can use these as per the spec to provide placeholders for custom markup.

Scoped CSS

Synergy provides lightweight, opt-in CSS scoping. Apply the scoped boolean attribute to a style tag within your components template, and all of the selectors will be prefixed with an additional type selector and hoisted up into the document head, giving you one style tag shared between all instances. This effectively stops your component styles from leaking out into the document, but doesn't stop anything from sneaking in, so you'll still need to be careful to use reasonable naming convention with your classes and attributes to keep them "unique" to your component and help to avoid accidental name clashes with other CSS on the page.
